# Demo Script — Resilient AgentOps Gateway

Target length: 90-120 seconds

## 0:00 - 0:15

Show the live app.

Narration:

```text
AI agents do not just need better answers. They need an operations layer. Resilient AgentOps Gateway shows what happened when an agent had to route, fall back, stop, or ask for human approval.
```

## 0:15 - 0:35

Show the top metrics.

Narration:

```text
At the top, the moderator or operator can see fallbacks used, approvals required, estimated cost, and blocked actions. This is the part that usually disappears inside logs.
```

## 0:35 - 0:55

Show TrueFoundry Challenge Fit.

Narration:

```text
The product is built around the TrueFoundry-style gateway story: routing, fallback, governance, and evidence. It is not claiming a live Gateway call yet. It is showing the cockpit that a real resilient-agent workflow needs.
```

## 0:55 - 1:20

Show the timeline.

Narration:

```text
Each step explains which model or tool path was tried, whether fallback was used, what risk was found, and whether human approval was required. A risky refund action is blocked instead of being hidden behind a final answer.
```

## 1:20 - 1:45

Show the handoff packet.

Narration:

```text
The handoff packet lets the next human or agent resume without guessing. It includes fallback count, blocked actions, approvals, cost, and the exact claim boundary.
```

## 1:45 - 2:00

Close on the boundary.

Narration:

```text
The next step is one real TrueFoundry Gateway smoke request and dashboard proof. Until then, this project keeps the claim boundary visible and machine-checkable.
```
